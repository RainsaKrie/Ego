# Ego

Ego 是一个正在设计中的、本地优先的个人 AI 工作台。

它的长期愿景是成为一个“个人 AI 操作系统”，但当前的 `v0.1` 会更聚焦：先做成一个支持 `BYOK` 的 AI 对话实验台，重点解决可观测性、Token 消耗追踪，以及 Prompt 与运行参数调优。

## 为什么要做 Ego

市面上大多数 AI 产品都会把真正关键的运行细节藏起来，比如：

- 当前请求到底塞了多少上下文
- 每一轮对话的 Token 是怎么涨上去的
- 模型参数到底怎样影响回答质量
- 不同记忆策略怎样在“效果”和“成本”之间做取舍

Ego 的出发点很简单：

做一个自己愿意长期使用的个人 AI 工作台，同时把它当成一个真正能帮助理解大模型运行机制的实践实验场。

## 产品方向

### 长期愿景

Ego 未来希望演进为个人 AI 操作系统，逐步具备：

- 以 AI 对话为主入口
- 可复用的个人知识层
- 后续可接入 Agent、工作流与外部工具
- 数据、配置、实验过程都尽量由本地掌控

### `v0.1` 产品形态

`v0.1` 会刻意收窄范围，先只聚焦：

- 多会话 AI 对话
- 基于 OpenAI 兼容接口的 `BYOK`
- `temperature`、输出长度等基础参数调节
- 每次请求的 Token、延迟、成本估算追踪
- 面向对话历史管理的基础记忆实验

当前暂不聚焦：

- 多 Agent 编排
- 工作流自动化
- 外部工具生态
- 长期知识库注入
- 大量面向特定厂商的深度适配
