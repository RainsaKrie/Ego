# Ego `v0.1` 实现优先级与里程碑

## 1. 文档定位

本文件基于当前已经确认的产品与架构规范，定义 `v0.1` 的实现优先级、依赖顺序和阶段里程碑。

它解决的核心问题是：

`先做什么，后做什么，哪些是首版必须先站住的骨架？`

大白话解释：
这份文档是在把“蓝图”变成“施工顺序”。

## 2. 当前 `v0.1` 的总目标复述

`Ego v0.1` 的目标不是“做一个功能很多的 AI 软件”，而是：

做一个本地优先、BYOK、可观测、可调优的 AI 对话实验台。

因此，首版真正必须站住的能力，不是越多越好，而是必须围绕这条主线：

- 对话能跑
- 参数能调
- 记录能落
- 观测能看
- 失败能解释

## 3. 实现优先级分层

`v0.1` 的实现项分为 4 层：

### P0：主链阻塞项

这些是“不做就不能叫 `v0.1`”的东西。

### P1：首版核心增强项

这些不是最底层阻塞，但属于首版体验的重要组成。

### P2：可延后增强项

这些有价值，但不应阻塞主链闭环。

### P3：明确不进入当前里程碑

这些内容需要明确写出来，避免首版边做边扩。

## 4. P0：主链阻塞项

以下内容进入 P0：

### 4.1 会话与消息主链

包括：

- 创建会话
- 切换会话
- 发送用户消息
- 接收助手消息
- 用户消息立即写入
- 成功后助手消息落地

### 4.2 请求记录主链

包括：

- 发请求前创建 `RequestRecord`
- 请求结束后更新状态
- 失败保留记录
- 取消保留记录

### 4.3 参数与记忆策略基础能力

包括：

- `model`
- `temperature`
- `top_p`
- `max_output_tokens`
- `memory_policy`

### 4.4 本地持久化主链

包括：

- 会话持久化
- 消息持久化
- 请求记录持久化
- usage 记录持久化
- 全局默认配置持久化
- 会话覆盖值持久化

### 4.5 全局设置与会话覆盖基础交互

包括：

- 新会话继承默认值
- 当前会话局部覆盖
- 偏离默认值提示
- 恢复默认环境

### 4.6 基础可观测性

包括：

- 最近一次请求的 token / latency / cost
- usage 来源区分
- 基础请求状态展示

### 4.7 敏感凭据主链

包括：

- `API Key` 的系统级安全存储

原因：

- 它属于主链可信度的一部分
- 不是后置增强项

## 5. P1：首版核心增强项

以下内容进入 P1：

### 5.1 会话分析页

包括：

- 请求总数
- 累计 Token
- 累计成本估算
- 平均延迟
- Prompt / Completion 构成
- Token 趋势
- 延迟趋势
- 弱化成本趋势

### 5.2 流式输出体验

包括：

- 流式文本临时显示
- 尾部光点 / 细小加载态
- 完成后转正式消息

### 5.3 失败恢复体验

包括：

- 失败提示靠近消息区
- 输入区轻量重试承接
- 按原条件重新生成

### 5.4 手动停止生成

包括：

- 生成中发送按钮切换为中止按钮
- 中止进入独立取消态

## 6. P2：可延后增强项

以下内容进入 P2：

### 6.1 更丰富的分析能力

例如：

- 参数阶段变化标记做得更精致
- 更复杂的趋势交互
- 更丰富的成本解释

### 6.2 更细的草稿体验

例如：

- 更完整的草稿恢复
- 草稿管理能力

### 6.3 更丰富的 provider 兼容层

例如：

- 超出 OpenAI-compatible 基线之外的适配细节

### 6.4 更丰富的安全交互

例如：

- 凭据管理体验增强
- 更完整的错误诊断提示

## 7. P3：明确不进入当前里程碑

以下内容明确不进入 `v0.1`：

- 多 Agent
- 工作流编排
- 长期知识库注入
- 工具市场
- 浏览器自动化
- 团队协作
- 云同步
- 复杂全局 BI 分析系统

## 8. 推荐里程碑拆分

`v0.1` 至少拆成 3 个实施里程碑：

### M1：主链可跑通

目标：

- 能创建会话
- 能发送消息
- 能拿到回复
- 能稳定写入消息与请求记录
- 能保存基础配置
- 能通过系统级安全存储管理 `API Key`

大白话解释：
M1 的目标不是“好看”，而是先让发动机点火。

### M2：实验台属性站住

目标：

- 参数调节生效
- 默认值与会话覆盖跑通
- 最近一次请求观测可见
- usage、latency、cost 基本可用
- 基础失败链路与重试可用
- 流式输出体验接入
- 手动停止生成接入

大白话解释：
M2 让它从“聊天能用”升级成“实验能用”。

### M3：分析与体验收口

目标：

- 会话分析页落地
- 存储与删除边界收口
- 分析与整体体验完成度提升

大白话解释：
M3 让它从“实验能用”升级成“比较像一个完整首版产品”。

## 9. 当前定案结论

`v0.1` 当前正式结论如下：

- `M1` 先打通消息、请求、存储与敏感凭据主链
- `M2` 再把实验与观测属性站住
- `M3` 最后补分析页和整体体验收口
- 会话分析页放在 `M3`
- 流式输出放在 `M2`
- 手动停止生成与流式输出一起进入 `M2`
- `API Key` 的系统级安全存储属于 `M1` 阻塞项
